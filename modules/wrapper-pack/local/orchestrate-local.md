# orchestrate-local

## Class

local

## Purpose

Coordinate multiple reviewed local roles or tools with visible task boundaries and no hidden routing.

## Allowed Use

- local task decomposition;
- local sequential or parallel plan notes;
- local synthesis planning.

## Approval Boundary

Ask before orchestration introduces deletion, destructive overwrite, elevated use, or non-reviewed external delegation.

## Rejection Cases

Reject use if the wrapper hides task routing, invents approvals, or obscures which tool or role receives each subtask.
