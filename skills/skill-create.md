# Skill: Skill Create

## Trigger

Use when:
- Ahmed runs `/skill-create` or `/skill-create --instincts` in a git repository
- Ahmed says "generate skills from this repo", "create skills from git history", or "analyze this codebase and make skill drafts"
- Starting work on an unfamiliar repo and Ahmed wants automated skill bootstrapping

Command: `/skill-create [--instincts]`

## What This Skill Does

Analyzes the git commit history of the current repository to discover recurring patterns — files that change together, commit message conventions, frameworks touched — then generates draft SKILL.md files for each pattern. Every draft starts at LOW confidence and requires human review before it is treated as a real skill.

This is not automatic skill adoption. It is automated first-draft generation.

## Process

### 1. Check git availability

```bash
git log --oneline -1
```

If this fails (not a git repo, git not installed, empty repo with no commits), stop:

```
Error: Cannot analyze git history.
  - This directory is not a git repository, or
  - git is not installed, or
  - The repository has no commits yet.

Run this skill from the root of a git repository with at least a few commits.
```

If there are fewer than 5 commits, warn but continue:

```
Warning: Only N commits in this repository. Pattern analysis may not be meaningful yet.
Continuing with what is available.
```

### 2. Collect raw history data

Run all three commands and capture their output:

```bash
# Last 100 commit messages (short form)
git log --oneline --no-merges -100

# All commit messages in full (for pattern extraction)
git log --format="%s" --no-merges

# Recent file change distribution
git diff --stat HEAD~20 HEAD

# Files changed per commit (last 100, for co-change analysis)
git log --no-merges -100 --name-only --format=""

# Commit message prefixes (for convention detection)
git log --format="%s" --no-merges | cut -d: -f1 | sort | uniq -c | sort -rn | head -20
```

If `HEAD~20` does not exist (fewer than 20 commits), use `git diff --stat $(git rev-list --max-parents=0 HEAD) HEAD` instead.

### 3. Identify recurring patterns

Analyze the collected data for three pattern types:

#### Pattern Type A: Co-changed files

Files that appear together in commits frequently suggest a domain or workflow:

- Count pairs of files that appear in the same commit across the last 100 commits
- Threshold: a pair must co-occur in at least 3 commits to be worth surfacing
- Cluster overlapping pairs into domains (e.g., `routes/`, `controllers/`, `tests/` all moving together → likely an API change workflow)

Example signal:
```
src/api/routes.js  + src/api/controllers.js  → 14 co-occurrences
tests/api/         + src/api/                → 11 co-occurrences
→ pattern: "API changes always require updating tests in tests/api/"
```

#### Pattern Type B: Commit message conventions

Scan commit message prefixes for established conventions:

- `fix:`, `feat:`, `refactor:`, `chore:`, `docs:`, `test:` → Conventional Commits in use
- `[WIP]`, `[HOTFIX]`, `[SKIP CI]` → custom prefixes
- Ticket numbers like `PROJ-123` or `#456` at the start → ticket-first workflow

Example signal:
```
feat: 23 commits
fix:  18 commits
refactor: 7 commits
→ pattern: "This repo uses Conventional Commits — all messages must start with feat:/fix:/refactor:/etc."
```

#### Pattern Type C: Languages and frameworks

Inspect file extensions and framework-specific paths touched in recent commits:

- `.ts` / `.tsx` → TypeScript project
- `migrations/` or `db/migrate/` → migration-based database
- `Dockerfile` or `.github/workflows/` → container/CI workflows
- `package.json` + `node_modules/` → Node.js
- `requirements.txt` or `pyproject.toml` → Python

Each detected stack is a candidate for a coding-standards skill.

### 4. Generate draft SKILL.md files for each pattern

For each identified pattern, generate one draft file.

File naming: `.claude/skills/generated/<kebab-case-pattern-name>.md`

Draft template:

```markdown
# Skill: <Title Derived from Pattern>

> DRAFT — generated by /skill-create on <date>. Confidence: LOW.
> Review and edit before promoting to a real skill.

## Trigger

Use when:
- <Situation inferred from commit history>
- <Second trigger if applicable>

## Pattern Source

Detected from git history:
- <Specific evidence: e.g., "routes.js and controllers.js co-changed in 14 of last 100 commits">
- <Second evidence line>

## Process

### 1. <Step inferred from pattern>

<Description inferred from commit messages and file changes>

### 2. <Next step>

<Description>

## Notes

- This draft was generated automatically. Verify each step against actual repo conventions.
- Confidence: LOW — needs human validation before relying on this skill.
- To promote: remove this header block and move to `.claude/skills/<name>.md`
```

**Title derivation rules:**

| Pattern type | Title format | Example |
|-------------|--------------|---------|
| Co-changed domain | `<Domain> Change Workflow` | `API Change Workflow` |
| Commit convention | `Commit Message Convention` | `Conventional Commits Convention` |
| Language/framework | `<Stack> Coding Standards` | `TypeScript Coding Standards` |
| Migration pattern | `Database Migration Workflow` | `Database Migration Workflow` |
| CI/CD pattern | `Deployment Workflow` | `Deployment Workflow` |

### 5. Create the output directory if needed

```bash
mkdir -p .claude/skills/generated/
```

If `.claude/` already exists at the project root, use it. If not, create `.claude/skills/generated/`.

Do not create the directory at the user's home or at a global path — this is always project-local.

### 6. Write the draft files

Write each generated SKILL.md to `.claude/skills/generated/`. Do not overwrite existing files with the same name — append `-2`, `-3` etc. if a file already exists.

### 7. If --instincts flag: generate instinct candidates for high-frequency patterns

Only for patterns with very high signal (co-occurrence >= 10, or commit convention present in >= 30% of commits):

Generate an instinct object for each, but do not write to the instinct store automatically. Instead, write them to `.claude/skills/generated/instinct-candidates.json`:

```json
[
  {
    "id": "generated-<timestamp>-<pattern-hash>",
    "trigger": "<inferred trigger from pattern>",
    "behavior": "<inferred behavior>",
    "outcome": "positive",
    "confidence": 0.25,
    "status": "pending",
    "source": "skill-create",
    "generated_at": "<today ISO date>",
    "note": "Auto-generated. Review before importing with /instinct-import."
  }
]
```

Confidence is always 0.25 for auto-generated instinct candidates — the lowest useful threshold. These are hypotheses, not confirmed patterns.

Print instructions for importing them after review:

```
Instinct candidates written to .claude/skills/generated/instinct-candidates.json
Review the file, then import reviewed entries:
  /instinct-import .claude/skills/generated/instinct-candidates.json
```

### 8. Report results

```
Skill drafts generated.

  Analyzed: 87 commits, 3 pattern types
  Generated: 4 skill drafts

  .claude/skills/generated/
    api-change-workflow.md         (co-change: routes+controllers+tests, 14 co-occurrences)
    conventional-commits.md        (commit prefix: feat/fix/refactor, 48 of 87 commits)
    typescript-coding-standards.md (language: .ts/.tsx in 91% of changed files)
    database-migration-workflow.md (co-change: migrations/+models/, 6 co-occurrences)

  Confidence: LOW on all drafts — review before use.

Next steps:
  1. Open each file in .claude/skills/generated/ and verify the steps
  2. Edit or delete steps that don't match reality
  3. Remove the DRAFT header block when satisfied
  4. Move the file to .claude/skills/<name>.md to promote it
```

## Example Generated Skill

What the generated `api-change-workflow.md` might look like:

```markdown
# Skill: API Change Workflow

> DRAFT — generated by /skill-create on 2026-04-19. Confidence: LOW.
> Review and edit before promoting to a real skill.

## Trigger

Use when:
- Modifying any file under src/api/routes/ or src/api/controllers/
- Adding a new API endpoint

## Pattern Source

Detected from git history:
- src/api/routes.js and src/api/controllers.js co-changed in 14 of last 100 commits
- tests/api/ changed in same commit as src/api/ in 11 of 14 cases

## Process

### 1. Update the route definition

Modify the relevant file in src/api/routes/.

### 2. Update the controller

Modify the corresponding file in src/api/controllers/.

### 3. Update or add tests

Based on history, API changes in this repo are consistently accompanied by changes in tests/api/.
Update the relevant test file.

## Notes

- This draft was generated automatically. Verify each step against actual repo conventions.
- Confidence: LOW — needs human validation before relying on this skill.
- To promote: remove this header block and move to .claude/skills/api-change-workflow.md
```

## How to Promote a Draft to a Real Skill

1. Open the draft in `.claude/skills/generated/<name>.md`
2. Read through every step — verify it matches how the repo actually works
3. Remove or rewrite any steps that are wrong or over-inferred
4. Delete the DRAFT header block at the top
5. Move the file: `mv .claude/skills/generated/<name>.md .claude/skills/<name>.md`
6. Test the skill by running it once manually and checking the output

Do not use a draft skill for automated or critical operations until it has been reviewed and promoted.

## Manual Git History Analysis Commands

```bash
# Most-changed files in last 100 commits
git log --no-merges -100 --name-only --format="" | sort | uniq -c | sort -rn | head -20

# Files that change together most often (co-change pairs — requires gitpython or manual)
git log --no-merges -100 --name-only --format="" | awk 'NF' | \
  python3 -c "
import sys
from itertools import combinations
from collections import Counter
commit, pairs = [], Counter()
for line in sys.stdin:
    line = line.strip()
    if line:
        commit.append(line)
    else:
        for pair in combinations(sorted(set(commit)), 2):
            pairs[pair] += 1
        commit = []
for (a, b), n in pairs.most_common(20):
    print(f'{n:3}  {a}  +  {b}')
"

# Commit prefix frequency
git log --format="%s" --no-merges | \
  python3 -c "
import sys, re
from collections import Counter
prefixes = Counter()
for line in sys.stdin:
    m = re.match(r'^(\w+)[:\(]', line.strip())
    if m:
        prefixes[m.group(1)] += 1
for prefix, count in prefixes.most_common(10):
    print(f'{count:4}  {prefix}')
"

# File extension distribution in recent changes
git diff --name-only HEAD~20 HEAD 2>/dev/null | \
  python3 -c "
import sys
from collections import Counter
exts = Counter()
for line in sys.stdin:
    parts = line.strip().rsplit('.', 1)
    if len(parts) == 2:
        exts['.' + parts[1]] += 1
for ext, n in exts.most_common():
    print(f'{n:4}  {ext}')
"
```

## Safe Behavior

- Read-only on the git repository — does not modify any source files.
- Only writes to `.claude/skills/generated/` — never modifies existing skills.
- Never writes to `~/.openclaw/instincts/` automatically, even with `--instincts`. Generated instincts go to a review file first.
- Never promotes a draft automatically — promotion requires Ahmed to manually move the file.
- If `.claude/skills/generated/` already contains files, appends with counter suffixes rather than overwriting.
- If git commands fail partway through, reports partial results and stops — does not write incomplete drafts.
