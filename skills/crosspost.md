# Skill: Crosspost

## Trigger

Use when distributing a piece of content across multiple platforms simultaneously — adapting the same core message for X/Twitter, LinkedIn, Threads, Bluesky, and other social networks, each in a format native to that platform.

## Process

### Phase 1 — Core Message Extraction

From the source content (article, announcement, insight, product update), extract:
- **Lead hook** — the most shareable single sentence
- **Core claim** — the main point in 1-2 sentences
- **Supporting points** — 3-5 key facts or arguments
- **Call to action** — what should the reader do? (Read, share, reply, sign up, etc.)
- **Link** — the URL to include (if any)

### Phase 2 — Platform Adaptation

Adapt for each target platform:

#### X / Twitter

```
Format: Thread (preferred for depth) or single tweet
Character limit: 280 per tweet
Style: Direct, punchy, hook in first tweet
Links: Include in last tweet to maximize reach

Tweet 1 (hook):      [The most arresting statement — make them stop scrolling]
Tweet 2-N (body):    [One point per tweet — short sentences, add line breaks for white space]
Last tweet (CTA):    [Link + clear ask — "Read the full post:" or "What do you think?"]
```

#### LinkedIn

```
Format: 150-300 word post, no header/title needed
Style: First-person, professional but conversational
Structure:
  Line 1: Hook sentence (no emojis, no clickbait)
  [blank line]
  Paragraph 1: Context / the insight
  [blank line]
  Paragraph 2-3: Key point expanded
  [blank line]
  Closing: Question to drive comments, or takeaway
  [blank line]
  Link (if applicable)
```

#### Threads

```
Format: Single post or short thread (3-5 posts)
Style: Casual, conversational — more personality than LinkedIn
Length: 200-500 chars per post
Tone: Like talking to a smart friend, not a professional audience
```

#### Bluesky

```
Format: Single post (300 chars) or thread
Style: Similar to Twitter — directness rewarded
Link cards: Paste URL last for auto-card
Note: No algorithm boost for likes/reposts yet — engagement is organic
```

### Phase 3 — Scheduling Considerations

| Platform | Best posting times (general) | Best days |
|---|---|---|
| X/Twitter | 8-10 AM, 12-1 PM local time | Tue-Thu |
| LinkedIn | 7-9 AM, 12 PM | Tue-Wed |
| Threads | Evening, 7-9 PM | Variable |

- Post in quick succession on all platforms (within 30 min) to capture the momentum window.
- Do not post identical text across platforms — platform algorithms detect and suppress it.

## Output Format

Produce platform-ready drafts, labeled and separated:

```markdown
## X/Twitter Thread

**Tweet 1:**
[text]

**Tweet 2:**
[text]

**Tweet 3 (CTA):**
[text + link]

---

## LinkedIn

[full post text]

---

## Threads

[post text]

---

## Bluesky

[post text]
```

## Constraints

- Each platform version must feel native — not a copy-paste with minor edits.
- Do not include the same URL in every tweet of a thread — it suppresses reach on X. Link only in the last tweet.
- Do not use excessive hashtags — 1-2 max on LinkedIn and Threads, 0-1 on X.
- Do not fabricate engagement predictions or promise virality.
